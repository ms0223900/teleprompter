# US-wordcount-02：編輯區顯示「總字數」與「去除標點後總字數」

**作為** 提詞器使用者  
**我想要** 在編輯稿件時即時看到總字數與去除標點後的字數  
**以便** 掌握稿長與實質文字量（不含標點）

**輸入格式**

- **須先完成**：**US-wordcount-00**（測試環境；若與 01 同迭代，至少與 01 一併完成）。
- **須先完成**：**US-wordcount-01**（`getManuscriptCharStats` 或等價 API）。
- 目前編輯模式下的稿件內容：與 `TelePrompter` 之 `text` 同步（含本機還原後之內容）。

**輸出格式**

- 在 **`mode === "edit"`** 的編輯畫面中（見 `components/TelePrompter.tsx` 之 `<textarea>` 區塊），新增一處**固定可見**的顯示區（例如 textarea 下方或右上角工具列），即時顯示兩項數字：
  - 文案建議：**「總字數：{n}」**
  - 文案建議：**「去除標點：{m}」**（或 **「不含標點字數：{m}」**，與產品用語一致即可）
- `text` 每次變更時重新計算（可 `useMemo` 依 `text` 快取），無需額外按鈕。
- 樣式與現有編輯區暗色／對比度協調，不遮擋輸入區。

**驗收條件**

- [ ] 僅在編輯模式顯示；播放模式不出現字數列（除非產品另有要求，預設不顯示）。
- [ ] 輸入、刪除、貼上文字時，兩個數字即時更新。
- [ ] 顯示數值與 `getManuscriptCharStats(text)` 結果一致。
- [ ] 空稿件時顯示 0／0（或 0 與 0，依文案）。
- [ ] 無 console 未捕捉錯誤；小螢幕下不擠壓 textarea 至不可用（基本響應式）。
- [ ] **整合測試**：行為驗證由 **US-wordcount-03** 負責；本 US 完成定義為「UI 與邏輯可用」，且最終須與 US-wordcount-03 一併通過 `npm run test:run`（若採嚴格 TDD，可先寫 03 再補本 US 至綠燈）。

**依賴關係**

- 須先完成：**US-wordcount-01**（邏輯與單元測試）。  
- **US-wordcount-03**（整合測試）依賴本 US 之可測行為；可並行採 TDD，見 US-wordcount-03。
