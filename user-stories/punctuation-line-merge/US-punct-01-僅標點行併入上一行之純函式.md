# US-punct-01：僅標點行併入上一行（純函式與單元測試）

**作為** 提詞器使用者  
**我想要** 逐字稿裡若某「單行」在去掉首尾空白後**僅含標點符號**（長度不限），該行在計算播放行時應併入上一行末尾  
**以便** 斷句／高亮不會單獨出現一整行只有「，。」「……」等標點，影響節奏與閱讀

**輸入格式**

- 單一參數：稿件全文 `string`（語意與 `TelePrompter` 之 `text` 相同）。
- 換行：以 `\n` 分段；若實作需相容編輯器貼上之 `\r\n`，建議先正規化為 `\n` 再處理（或於測試中覆蓋 `\r\n` 案例）。

**輸出格式**

- 新增模組（建議 `lib/mergePunctuationOnlyLines.ts` 或等價路徑），至少匯出：
  - `normalizeManuscriptLinesForPlayback(text: string): string`
- **語意**：回傳值與輸入為同一「稿件」內容，僅將「應併入上一行」的標點行合併；合併時將該行字元**直接接在上一行末尾**（中間不插入額外換行；上一行尾與該行頭之間不強制插入或刪除空白，建議規則見下）。
- **「僅標點」定義**（建議與專案既有「去標點」語意對齊）：
  - 對一行先做 `trim()`；若結果為空字串，**不適用**本規則（空行維持由下游既有邏輯過濾，本函式可選擇保留換行結構或先剔除全空行—**須在實作與測試中固定一種行為**）。
  - 若 `trim()` 後之字串**每個字元**皆屬於 Unicode **標點**類別，建議使用 `RegExp` 之 **`\p{P}`**（需 `u` 旗標）逐字檢驗；通過則視為「僅標點行」。
- **合併規則**：
  - 由上而下掃描「邏輯行」；若當前行為僅標點行且**存在上一行**，則將該行 `trim()` 後之內容**接在上一行末尾**，並移除該獨立行。
  - 若稿件**第一個非空邏輯行**即為僅標點行（無上一行可併），則**保留該行**為獨立一行（不刪除、不與不存在的上一行合併）。
  - **連續多行**僅標點：應逐一併入當時的「上一行」，直到遇到第一行含非標點文字為止（例：`你好\n，\n。` → 一行 `你好，。`）。

**驗收條件**

- [x] `normalizeManuscriptLinesForPlayback` 為純函式：不依賴 `window`／DOM；可被單元測試與 `TelePrompter` 共用。
- [x] **案例**：`"第一句\n，\n第二句"` → 合併後語意等同 `"第一句，\n第二句"`（具體字串以實作對「空行／換行保留」的約定為準，但播放時「，` 不得單獨成行）。
- [x] **案例**：`"A\n，。！\nB"` → `，`、`。`、`！` 皆併入 `A` 所屬行之末（或合併後之單行），`B` 仍為新行。
- [x] **案例**：第一行僅 `，`（無前置文字行）→ 仍保留至少一行內容為 `，`（或 trim 後之 `，`），不拋錯。
- [x] **案例**：含 `\p{P}` 可涵蓋之英文標點與全形標點（至少各一則測試）。
- [x] 同目錄或慣例路徑新增 **`lib/mergePunctuationOnlyLines.test.ts`**（或 `*.spec.ts`），上述行為以測試鎖定。
