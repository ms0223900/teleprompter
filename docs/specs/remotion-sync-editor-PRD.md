## 產品需求文件 (PRD) - Remotion Sync Editor 模組

### 1. 產品願景與目標 (Vision & Goals)

#### **我想達成的事情 (The "Why")**
* **消除認知負荷：** 讓我在撰寫逐字稿的同時，就能「看到」時間軸的分配，不再需要在 Remotion 代碼與文字稿之間反覆盲測。
* **建立單一事實來源 (Single Source of Truth)：** 提詞器不再只是讀稿工具，而是整個影片的**時序控制器**。劇本調整，動畫參數自動同步。
* **精準分鏡設計：** 透過「相對經過時間」，我能準確判斷某個 Step 是否話太多太乾，或話太短導致動畫演不完。

#### **成功指標 (Success Metrics)**
* **修改次數降低：** 減少 50% 以上在 Remotion 中手動微調 `durationInFrames` 的時間。
* **錄製流暢度：** 錄音時的語速與動畫切換點達成 90% 以上的體感對齊。

---

### 2. 使用者流程 (User Journey)
1.  **編寫期：** 在提詞器輸入文字，並在關鍵轉折處插入 `[label]`。
2.  **調校期：** 側欄即時顯示該段落的 Frames，若秒數太長，我會當場修改文字精簡度。
3.  **串接期：** 一鍵匯出 JSON，放入 Remotion 專案。
4.  **渲染期：** Remotion 動畫組件根據 JSON 內的 `startFrame` 與 `durationFrames` 自動排列。

---

### 3. 完整功能需求 (Functional Requirements)

#### **A. 編輯器增強：動態標籤系統**
* **[REQ-1] 自訂標籤：** 支持 `[labelName]` 格式，不限名稱，僅限英數字與底線。
* **[REQ-2] 語法突顯：** 在編輯器中以不同顏色顯示標籤（如藍色或粗體），與一般文字區隔。

#### **B. 時序運算引擎 (Core Engine)**
* **[REQ-3] 基礎換算：** 基於 WPM 與 30 FPS 固定頻率，計算標籤間的總幀數。
* **[REQ-4] 停頓權重：** 自動辨識標點符號並增加 Buffer Frames（補償真人呼吸與停頓）。
* **[REQ-5] 即時反饋：** 用戶每打一個字，側欄的秒數/幀數需即時更新（Debounced Update）。

#### **C. 數據出口 (Data Interface)**
* **[REQ-6] Label 數據清單：** 顯示包含 `label`、`預計秒數`、`總幀數` 的列表。
* **[REQ-7] 標準 JSON 匯出：** 欄位必須包含 `label` (自訂名稱)、`startFrame` (累積起點)、`durationFrames` (區段長度)。

### 4. User Stories 拆解預覽 (User Stories Backlog)

為了讓你後續好開發，我幫你預拆了幾個關鍵的 User Stories：

| 編號 | 角色 | 想要做什麼 (Action) | 以達成什麼 (Benefit) |
| :--- | :--- | :--- | :--- |
| **US-1** | 創作者 | 我想在文字中插入 `[Step1]` 標記 | 以便定義動畫切換的物理位置。 |
| **US-2** | 創作者 | 我想在側欄看到每個標籤間的 Frames 總數 | 以便我決定 Remotion 動畫要設計得複雜還是簡單。 |
| **US-3** | 創作者 | 我想調整 WPM 數值 | 以便適應我今天不同的說話語速，並看見時間軸的連動變化。 |
| **US-4** | 開發者 | 我想匯出包含 Label 與 Frames 的 JSON | 以便讓 AI 能直接把這組數據塞進 Remotion 的 `props`。 |
| **US-5** | 創作者 | 我想讓系統自動計算標點符號的停頓 | 以免實際錄音時因為語氣停頓，導致動畫跑得比說話快。 |

---

### 5. 技術約束與假設 (Technical Constraints)
* **環境：** Web-based App (React)。
* **FPS：** 固定 30（第一階段不需做成可選，減少變數）。
* **語言：** 需同時兼容中英文計數邏輯（中文算字數，英文算單詞數）。

---

### PM 大師的最後叮嚀：
這份 PRD 的核心競爭力在於 **「預測性」**。
在開發時，建議你先實作 `US-2`（顯示 Frames），因為這對你的「決策影響力」最大。當你發現寫了兩句話卻要花 10 秒（300 frames）時，你就會立刻決定在那裡多加一個動畫 step 或修減文字。
