# US-01：標籤語法解析器

**作為** 創作者
**我想要** 在逐字稿中以 `[labelName]` 格式插入標籤
**以便** 定義動畫切換點的物理位置

## 輸入格式
- 文字稿純文字字串（含中英文、標點、換行）
- 標籤語法規則：`[labelName]`，名稱限英數字與底線（`[A-Za-z0-9_]+`）

## 輸出格式
- 純函式 `parseLabels(text: string): ParsedSegment[]`
- `ParsedSegment` 結構：
  ```ts
  type ParsedSegment = {
    label: string;        // 標籤名稱
    startIndex: number;   // 標籤在原文的起始字元位置
    endIndex: number;     // 標籤結束位置
    contentBefore: string;// 此標籤「之前」到上一個標籤（或開頭）之間的文字內容（不含標籤本身）
  };
  ```
- 不合法的標籤（如 `[中文]`、`[ab c]`）視為普通文字，不回傳。

## 驗收條件
- [ ] 能正確抽取單一合法標籤
- [ ] 能按出現順序抽取多個標籤
- [ ] 非法字元的方括號內容不會被當成標籤
- [ ] 未閉合的 `[` 或 `]` 不會造成解析錯誤
- [ ] `contentBefore` 不包含標籤本身
- [ ] 提供單元測試覆蓋以上情境

## 依賴關係
- 無（基礎工具函式）
