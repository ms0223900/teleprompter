# US-07：側欄標籤數據清單

**作為** 創作者
**我想要** 在側欄看到每個標籤的預計秒數與總幀數
**以便** 判斷 Remotion 動畫要設計得複雜還是簡單

## 輸入格式
- `LabelTiming[]`（來自 US-06）

## 輸出格式
- 側欄 React 元件，表格或清單形式，顯示欄位：
  - `label` 名稱
  - `words`（字/詞數）
  - `seconds`（小數一位）
  - `frames`（整數）
  - `startFrame`（累積起點）

## 驗收條件
- [ ] 每個合法標籤顯示一列
- [ ] 順序與標籤在文稿中出現順序一致
- [ ] 秒數 > 警示閾值（例如 10 秒）時以視覺樣式提示
- [ ] 秒數 < 警示閾值（例如 1 秒）時以視覺樣式提示
- [ ] 空列表時顯示「使用說明」區塊（見下方規格），非僅單行提示
- [ ] RWD：在小螢幕下不破版
- [ ] 側欄常駐「?」icon，hover/click 展開使用說明 popover，不論列表是否為空皆可查閱

## 使用說明（側欄內建）

當側欄**無任何合法標籤**時，清單位置直接顯示完整說明區塊；**有標籤時**，說明收合至側欄頂端「?」icon 後，hover/click 以 popover 形式呈現。

說明區塊需涵蓋：

1. **功能定位**
   > 標籤（Label）是分鏡記號，用於在逐字稿中標示「動畫切換點」。寫稿同時就能看到每一段 Step 的預計秒數與幀數，並可匯出為 Remotion 可直接消費的 JSON。

2. **語法規則**
   - 格式：`[labelName]`
   - 名稱限制：英數字與底線（`A-Z`、`a-z`、`0-9`、`_`）
   - 合法範例：`[Intro]`、`[Step_1]`、`[outro_2]`
   - 非法範例：`[中文]`、`[has space]`、`[foo-bar]`、`[]`

3. **實際範例**（以 code block 呈現）
   ```
   [Intro]
   今天要介紹三個重點。
   [Step_1]
   第一個重點是……
   [Step_2]
   第二個重點是……
   [Outro]
   以上就是今天的分享，謝謝大家。
   ```

4. **行為說明**
   - 編輯模式：標籤以藍色底/粗體突顯，方便一眼辨識。
   - 播放模式：標籤**不會出現在畫面上**，只作為切段依據。
   - 側欄：每個標籤顯示一列，呈現字數、秒數、幀數與累積起始幀。
   - 匯出：點「匯出 JSON」即可拿到 `{ label, startFrame, durationFrames }` 給 Remotion 使用。

5. **快速開始**
   - 可直接使用側欄的「+ 插入標籤」按鈕（見 US-10），會自動編號並插入於游標處。
   - 也可手動輸入 `[labelName]`。
